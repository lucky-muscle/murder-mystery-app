> 5人用マーダーミステリーをWeb上で進行するためのゲーム支援ツールです。

> ※ネタバレ防止のため、シナリオ本文や調査カードの具体的な内容は非公開にしています。現在は、ゲーム進行管理や画面遷移などのシステム面のみ確認できる状態です。

## 概要

本プロジェクトは、マーダーミステリーの進行をWeb上で管理するために制作したアプリケーションです。

キャラクター選択、作品概要の表示、調査フェーズの進行、調査カードの表示、推理・投票、エンディング表示までを一連の流れとして実装しました。

マーダーミステリーでは、プレイヤーごとに閲覧できる情報や進行中に開示される情報が異なるため、ゲームフェーズや選択キャラクターに応じて画面表示を切り替える構成にしています。

## 制作目的

紙の資料やGMの口頭説明に依存しやすいマーダーミステリーの進行を、Webアプリ上で整理して管理できるようにすることを目的としました。

特に、以下の点を意識して制作しました。

* ゲームの進行状況をフェーズとして管理すること
* プレイヤーごとに表示する情報を切り替えること
* 調査カードをデータとして扱い、追加・修正しやすくすること
* ゲーム開始から終了までの流れを1つのアプリで確認できるようにすること

## 主な機能

* キャラクター選択機能
* キャラクターの重複選択防止
* ゲームフェーズごとの画面切り替え
* 調査カードの表示
* 調査マップの表示
* 推理・投票画面
* エンディング表示
* 作品概要・進行説明の表示

## Reactを使用した部分

Reactを用いて、タイトル画面、キャラクター選択画面、調査画面、投票画面、エンディング画面などをコンポーネントとして分割しました。

また、現在のゲームフェーズ、選択中のキャラクター、表示中の調査カードなどを状態として管理し、状況に応じて表示する画面を切り替えています。

これにより、マーダーミステリーの複雑な進行を、1つのWebアプリ上で整理して扱えるようにしました。

## 実装上の工夫

### ゲーム進行をフェーズとして管理

ゲームの進行を、キャラクター選択、作品概要、調査、推理、投票、エンディングなどのフェーズに分けて管理しました。

現在のフェーズに応じて表示する画面を切り替えることで、プレイヤーが次に行う操作を分かりやすくしています。

### 調査カードをデータとして管理

調査カードの内容を画面に直接書くのではなく、データとして管理する構成にしました。

これにより、カードの追加や修正をしやすくし、ゲーム内容の調整に対応しやすい構成にしています。

### プレイヤーごとの情報表示を意識

マーダーミステリーでは、プレイヤーごとに見えてよい情報が異なります。

そのため、キャラクター選択やゲームフェーズに応じて表示内容を切り替える設計にし、ネタバレを防ぎながら進行できるようにしました。

## はじめ方

### 前提条件

* Node.js
* npm

### セットアップ

```bash
git clone https://github.com/lucky-muscle/murder-mystery-app.git
cd murder-mystery-app
npm install
npm run dev
```

開発サーバー起動後、ターミナルに表示されるURLにアクセスしてください。

## ビルド

```bash
npm run build
```

## デプロイ

Firebase Hostingを使用しています。

```bash
firebase deploy --only hosting
```
